# ADR-0001: ビルドレス・JSON専用構成

## ステータス
採用

## 確信度
高（CLAUDE.md に「ソースコード・ビルド・テストは無い。`themes/*.json` と `package.json` がすべて。」と明示されており、リポジトリ構成からも確認できる）

## コンテキスト
VS Code カラーテーマ拡張機能を作成するにあたり、実装言語・ビルドパイプラインの有無を選択する必要があった。
VS Code のカラーテーマは `package.json` + テーマ定義 JSON だけで動作する宣言的な仕組みであり、TypeScript / JavaScript のソースコードは不要。

## 決定
ソースコード・ビルドステップ・テストを持たず、`themes/*.json` と `package.json` のみで拡張機能を構成する。

## 理由
VS Code カラーテーマの仕様上、色定義は JSON で完結し、コンパイルや実行時処理が一切不要である。
TypeScript 等を導入するとビルド設定・依存管理・型検査の維持コストが生じるが、JSON 専用構成ならファイルを直接編集するだけで変更が完結する。

## 捨てた選択肢
- **TypeScript 拡張機能（コードあり）** → テーマに動的な挙動（設定連動カラー切替等）は現時点で不要なため過剰。JSON で要件を満たせる。
- （その他の選択肢については調査時点で痕跡なし）

## 影響範囲
- `themes/*.json`：テーマ定義の実体。ここだけ編集すれば機能変更が完結。
- `package.json`：拡張マニフェスト兼テーマ登録リスト。テーマ追加時は両方を更新する。
